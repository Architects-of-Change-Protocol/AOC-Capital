# Governance Gate Failure Inventory v2

Date: 2026-06-16  
Scope: classification-only inventory for the currently failing governance gate tests.  
Constraint honored: no production code and no test code were modified for this inventory.

## Executive summary

A fresh `npm test` run in `/workspace/pmfreak` reported **37 failing tests** out of 2,743 tests (`2706` pass, `37` fail). The request referenced 38 remaining failing tests; this inventory covers every failing test observed in the current checkout. The separate governance command previously observed outside this inventory (`npm run check:governance`) also fails at `check:package-exports` because AOC `dist` build artifacts are missing, but that is a command failure rather than an individual `node:test` test failure and is therefore listed in the remediation notes rather than counted in the table below.

The failures cluster into six areas:

1. **Runtime/authority boundary enforcement**: two failures are real architectural violations around imports that bypass the runtime-consumer boundary.
2. **Workspace runtime UI contracts**: many tests assert older direct localStorage/awakening/imprint/navigation surfaces while current code uses `bootstrapRuntimeState`, `runtimePersistence`, and a compressed/canonical workspace flow.
3. **Navigation semantic contracts**: current navigation labels and grouped shell rendering diverge from tests expecting a collapsed hierarchy (`Workspace`, `Summary`, `Execution`, etc.).
4. **Upload/evidence contracts**: tests assert legacy in-route extraction and a copilot upload UI that no longer exists because `/copilot` redirects to `/workspace` and upload processing is queued through the evidence processor.
5. **Operational memory contracts**: one test still expects the removed v1 append/extraction path in upload/copilot routes.
6. **Algorithmic/runtime behavior**: one lifecycle event naming mismatch and one pressure-weight cap assertion mismatch need contract reconciliation.

## Recommended remediation plan

### Phase 1 — Fix real architecture violations first

1. **Runtime-consumer authority boundary**
   - Fix app/API routes that import `@/lib/security/access-guards` directly when they participate in runtime authority decisions.
   - Move allowed authority interactions through `@/aoc/runtime-consumer` or provide a formal exception list if a route only needs an error type.
2. **Runtime-consumer import boundary**
   - Move or re-export live federation event types through an approved runtime-consumer boundary, or extend the allowed boundary list only after explicit design review.

### Phase 2 — Resolve navigation/workspace product contract drift

1. Decide whether the canonical navigation taxonomy is the test taxonomy (`Workspace`, `Summary`, `Execution`, `Executive`, `Portfolio`, `Projects`, `Upload`, `Settings`) or the current implementation taxonomy (`Create PMO`, `Create Project`, `Overview`, `Delivery Status`, `Leadership View`, `Project Controls`, etc.).
2. If the tests are authoritative, fix `navigation-hierarchy.ts` and `operational-shell.tsx`.
3. If the current UI copy is intentional, replace `navigation-collapse.test.mjs`, `legacy-surface-demotion.test.mjs`, and workspace compression contracts with approved current-copy contracts.

### Phase 3 — Replace stale runtime-state contracts

1. Update tests that assert direct localStorage helpers (`persistAwakeningState`, `loadImprintState`, `persistImprintState`, `isRuntimeValidationEnabled`) to assert the current `bootstrapRuntimeState` and `runtimePersistence` abstractions.
2. Remove or replace tests that assert dormant telemetry cards, direct awakening event subscriptions in operational shell, adaptive prompt starters, imprint-driven lens sorting, and panel rendering if those surfaces were intentionally removed.
3. If those surfaces were not intentionally removed, restore implementation and remove unused imports currently left in `workspace-conversation-shell.tsx`.

### Phase 4 — Replace upload and operational memory v1 contracts

1. Update upload UI tests to inspect the canonical workspace upload surface rather than the redirected copilot page.
2. Replace the synchronous `extractWithTimeout` upload-route contract with a queued/background extraction contract around `processEvidenceInBackground` and the evidence processor.
3. Replace legacy operational memory v1 append/extraction route assertions with canonical evidence/vault ingestion assertions.

### Phase 5 — Correct narrow algorithm/event-contract mismatches

1. Align persistent lifecycle event naming (`approval_granted` vs `approval_approved`) with the canonical lifecycle event type list.
2. Update `computePressureWeight` tests to acknowledge saturation at `1.0`, because strict monotonic increase cannot hold after the cap.
